* 内存与数据存储相关
- 比特(bit)：最小的单位，每比特只能存储0或1两种状态（二进制位）
- 字节(byte)：1字节等于8比特，常用单位（数据处理基本单位），缩写成 =B=
  - B,KB,MB,GB,TB 之间的倍率均为1000，往上还有 PB,EB 等个人不常用单位
  - B,KiB,MiB,GiB,TiB 之间的倍率均为1024

- 对于数组int32_t arr[10]，sizeof(arr)的结果是40（即整个数组占用的空间），而
- 对于指针int32_t *ptr=arr，sizeof(ptr)的结果是8（指针本身占用的大小）。

对于各变量大小，其能存储的最大数据量为2的n次方（n为数据的比特数，即字节数的8倍）。
具体地说，1字节最多存储2^(1*8)=256个不同的数据（两个十六进制数），即从0x00到0xFF

特别的，结构体内部处于性能原因会出现填充（在变量间插入未使用的空间更改地址位置提高执行效率）
现象，导致结构体总大小比各成员的大小总和大

对于大小端模式，对于 =(int)0x87654321= ，其数据存储分别如下：
- 大端模式（人类易读）：
  #+begin_src mono-text
      0x100    0x101    0x102    0x103        地址
    +--------+--------+--------+--------+
    |  0x87  |  0x65  |  0x43  |  0x21  |     数据
    +--------+--------+--------+--------+
      ↑高字节                    ↑低字节
    （低地址）                  （高地址）
  #+end_src
- 小端模式（x86,ARM等）（机器易读写）：
  #+begin_src mono-text
      0x100    0x101    0x102    0x103       地址
    +--------+--------+--------+--------+
    |  0x21  |  0x43  |  0x65  |  0x87  |    数据
    +--------+--------+--------+--------+
      ↑低字节                    ↑高字节
    （低地址）                  （高地址）
  #+end_src
- 区别：一多字节变量类型内部的数据排布方向与地址变化方向是否相同\\
  大小端各有优点，对于小端来说强制类型转换时不需要调整内存数据位置
  （例如u32(0x87654321)转u16(0x4321)直接把高地址砍掉即可）

#+begin_quote
补码是用来储存负数的一种方法，以int8_t为例：
- =00 00 00 01= 表示1
- =11 11 11 11= 表示-1 (补码)
- 进行相加操作时刚好溢出形成0，无须额外逻辑
- 补码=(~(int)i)+1 ,即将其按位取反然后加1
- 在此设定下，最高位就可用于判断正负数
#+end_quote

** 文本编码
常用编码：
- ASCII: 数字、英文字母、英文标点等的编码，每个字符占1字节（8比特）
- GB2312,GBK：国内爱用的编码，兼容ASCII，汉字用两字节表示
- UTF-8: 使用较为广泛的编码，兼容ASCII，与GBK等冲突，汉字一般使用三字节表示

个人经验而言，国内Windows电脑使用gbk编码较多，其他设备多是使用utf8编码。utf8能编码的字符比gbk多

相关库：iconv(编码转换)

* C语言语法相关

** 操作顺序
- 对于表达式 =(a,b,c)= ，a,b,c会顺次执行，最后返回c的值
- 自增自减的 =a++= 与 =++a= 两种形式的区别在于将它们传参时是返回操作前或操作后的值
- 对于表达式 =(a && b)= ，b会在a非0时执行，最后返回bool值而非a或b的计算值（别python病犯了）

** 复杂类型定义与理解
对于复杂类型（函数指针、指针、数组复合结构），优先找括号，其次向右看，最后向左
- E.g. =char *ptr[10]= 是指针数组（先是长度为10的数组，再是内部元素类型为指针）
  (sizeof(ptr)=80, sizeof(*ptr)=8)
- E.g. =int (*ptr)[10]= 是数组指针（先是指针，再是其指向内容为长度10的int型数组）
  (sizeof(ptr)=8, sizeof(*ptr)=10)
- E.g. =int (*ptr[10])(char*,char*)= 是函数指针数组（基本等同于第一个例子只不过内部元素为函数指针）
  使用类似 =ptr[num]("...", "...");= 进行函数调用
#+begin_quote
观察下来汉语里从左到右顺序对应的类型顺序大概是从里到外（从访问角度来看的里外），
优先级从低到高
#+end_quote

问ai：带有参数且返回 (=函数指针= 数组)指针的 =函数指针= 该怎么定义？

方法一：使用typedef逐步定义类型
#+begin_src c
  // 步骤1：定义基础函数指针类型（指向返回 int，接受两个 int 的函数）
  typedef int (*FuncPtr)(int, int);

  // 步骤2：定义指向“包含5个 FuncPtr 的数组”的指针类型
  typedef FuncPtr (*ArrayPtr)[5];

  // 步骤3：定义指向函数的指针，该函数接受 (float, char)，返回 ArrayPtr
  typedef ArrayPtr (*MyFuncPtr)(float, char);

  // 使用类型别名定义变量
  MyFuncPtr fp;

#+end_src
方法二：直接声明（复杂，但展示语法规则）
#+begin_src c
  int (*(*(*fp)(float, char))[5])(int, int); /* 说明：
  int (*                        )(int, int); -> 函数指针
                             [5]  -> 数组
        (*                  )     -> 指针(下面的函数指针的返回值)
          (*  )(float, char)      -> 函数指针(用fp(0.1,'6')调用)
            fp
  自下而上为从外到内 */
#+end_src

** 宏定义
特殊宏：
- =__LINE__= 代码当前行数（应用参照上面的一个示例）
- =__FILE__= 编译时的代码文件名
- =__FUNCTION__= 编译时的代码所在的函数名
特殊字符：
- =#= 能将宏传入的参数转换成字面量字符串(用引号括起来并对内部的特殊字符转义)
  #+begin_src
  #define marco(x) printf("%s = %d\n", #x, (x))
  void main(){ int yee=2; marco(++yee); }
  #+end_src
  打印出来的结果应该是 =yee++ = 2=
- =##= 衔接符号（两不同符号合并在一起变成第三个符号）

* 系统调用相关
- 执行程序（直接替换进程）

  - execve: 需要指定环境变量字符串数组

  一般而言，execv的argv[0]需要指定为程序名（路径名），可作为多调用程序使用
  （ln -s a.out ls，执行./ls时程序获取的argv[0]名称为ls，作为busybox这类工具集执行不同程序的判断基础）
