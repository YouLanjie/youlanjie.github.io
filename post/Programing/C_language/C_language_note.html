1. 内存与数据存储相关

  • 比特(bit)：最小的单位，每比特只能存储0或1两种状态（二进制位）
  • 字节(byte)：1字节等于8比特，常用单位（数据处理基本单位），缩写成 B
    • B,KB,MB,GB,TB 之间的倍率均为1000，往上还有 PB,EB 等个人不常用单位
    • B,KiB,MiB,GiB,TiB 之间的倍率均为1024

1.1. 各类型的大小

根据不同平台具体大小可能会有差异，可以下载编译这个程序查看结果，参考如下：

|    sizeof(TYPE) | SIZE(bytes) |
|-----------------|-------------|
|            char | 1           |
|           short | 2           |
|             int | 4           |
|           float | 4           |
|            long | 8           |
|          double | 8           |
|-----------------|-------------|
|   unsigned char | 1           |
|  unsigned short | 2           |
|        unsigned | 4           |
|    unsigned int | 4           |
|   unsigned long | 8           |
|-----------------|-------------|
|          int8_t | 1           |
|         int16_t | 2           |
|         int32_t | 4           |
|         int64_t | 8           |
|-----------------|-------------|
|         uint8_t | 1           |
|        uint16_t | 2           |
|        uint32_t | 4           |
|        uint64_t | 8           |
|-----------------|-------------|
|           _Bool | 1           |
|            bool | 1           |
|          size_t | 8           |

其中int8_t等类型应当是长度固定/明确的，其中的数字为类型的大小（以比特位单位）

注：多数情况下sizeof是一个编译时展开的运算符（类似于宏）而它对数组和指针的处理不同：对于数组int32_t arr[10]，sizeof(arr)的结果是40（即整个数组占用的空间），而对于指针int32_t *ptr=arr，sizeof(ptr)的结果是8（指针本身占用的大小）。但若是变长数组int arr[size]则会在运行时进行求值。

对于各变量大小，其能存储的最大数据量为2的n次方（n为数据的比特数，即字节数的8倍）。具体地说，1字节最多存储2^(1*8)=256个不同的数据（两个十六进制数），即从0x00到0xFF

1.2. 内存模型

由于这段内容比较枯燥故借助ai生成内容讲解

双击字符画切换字体以对齐

内存整体布局：

+----------------------------------+  高地址
|           栈 (Stack)             |
|   向下增长 (地址递减)            |
|  (函数调用、局部变量、返回地址)  |
+----------------------------------+
|                                  |
|            空闲内存              |
|                                  |
+----------------------------------+
|           堆 (Heap)              |
|   向上增长 (地址递增)            |
|  (动态分配: malloc/new)          |
+----------------------------------+
|         数据段 (Data)            |
|  已初始化全局/静态变量 (.data)   |
|  未初始化全局/静态变量 (.bss)    |
+----------------------------------+
|         代码段 (Text)            |
|   只读，存放机器指令             |
+----------------------------------+  低地址

对于大小端模式，对于 (int)0x12345678 ，其数据存储分别如下：

  • 大端模式（人类易读）：

       0x100     0x101     0x102     0x103    地址
+--------+--------+--------+--------+
|  0x12  |  0x34  |  0x56  |  0x78  |     数据
+--------+--------+--------+--------+
   ↑高字节                        ↑低字节
（低地址）                      （高地址）

  • 小端模式（x86,ARM等）（机器易读写）：

     0x100     0x101     0x102     0x103     地址
+--------+--------+--------+--------+
|  0x78  |  0x56  |  0x34  |  0x12  |    数据
+--------+--------+--------+--------+
   ↑低字节                        ↑高字节
（低地址）                      （高地址）

  • 区别：一多字节变量类型内部的数据排布方向与地址变化方向是否相同
    类别：机器看字是顺着地址123456一路看下去的，将较小的数（较常出现）写在低地址提高读写速度

对于类型转换，

符号拓展：

char c = -1;      内存（小端，1 字节）：
                +--------+
                |  0xFF  |
                +--------+

                      ↓ 符号扩展（高位填充符号位）

int i = c;        内存（小端，4 字节）：
                +--------+--------+--------+--------+
                |  0xFF  |  0xFF  |  0xFF  |  0xFF  |
                +--------+--------+--------+--------+
                 低地址                          高地址

二进制解释：11111111 11111111 11111111 11111111 → 补码 -1

若c为unsigned char，则i会变为255(0x000000FF)

截断：

int x = 0x12345678;    内存（小端，4 字节）：
                      +--------+--------+--------+--------+
                      |  0x78  |  0x56  |  0x34  |  0x12  |
                      +--------+--------+--------+--------+
                       低地址                          高地址

                            ↓ 截断：只保留低 2 字节（0x5678）

short y = (short)x;    内存（小端，2 字节）：
                      +--------+--------+
                      |  0x78  |  0x56  |
                      +--------+--------+
                       低地址  高地址

值 = 0x5678（十进制 22136），若 short 为有符号则为正数。

有无符号转变：

int s = -1;           内存（小端，4 字节）：
                     +--------+--------+--------+--------+
                     |  0xFF  |  0xFF  |  0xFF  |  0xFF  |
                     +--------+--------+--------+--------+

                           ↓ 位模式完全相同

unsigned int u = (unsigned int)s;
                     +--------+--------+--------+--------+
                     |  0xFF  |  0xFF  |  0xFF  |  0xFF  |
                     +--------+--------+--------+--------+

解释：s = -1 (补码)      u = 4294967295

补码是用来储存负数的一种方法，以int8_t为例：

  • 00 00 00 01 表示1
  • 11 11 11 11 表示-1 (补码)
  • 进行相加操作时刚好溢出形成0，无须额外逻辑
  • 补码=(~(int)i)+1 ,即将其按位取反然后加1
  • 在此设定下，最高位就可用于判断正负数

整数转小数（需要额外运算）（为方便观看这里高低地址示意方向反过来了）：

int n = 123;          内存（小端，4 字节）：
                     +--------+--------+--------+--------+
                     |  0x00  |  0x00  |  0x00  |  0x7B  |
                     +--------+--------+--------+--------+
                      高地址                          低地址
二进制（整数）：00000000 00000000 00000000 01111011

                              ↓ 转换运算（不是简单拷贝）

float f = (float)n;   内存（小端，4 字节）：
                     +--------+--------+--------+--------+
                     |  0x42  |  0xF6  |  0x00  |  0x00  |
                     +--------+--------+--------+--------+
                      高地址                          低地址
二进制（IEEE 754）：01000010 11110110 00000000 00000000
                    ↑符号(0) ↑指数(133-127=6)  ↑尾数(1.921875)
实际值 = 1.921875 × 2^6 = 123.0

小数转整数：

float f = 123.456f;    内存（小端，4 字节）：
                      +--------+--------+--------+--------+
                      |  0x42  |  0xF6  |  0x7A  |  0xE9  |
                      +--------+--------+--------+--------+
                       高地址                          低地址

                              ↓ 转换（向零截断）

int n = (int)f;        内存（小端，4 字节）：
                      +--------+--------+--------+--------+
                      |  0x00  |  0x00  |  0x00  |  0x7B  |
                      +--------+--------+--------+--------+
                       高地址                          低地址

值 = 123（小数部分丢失）

1.3. 文本编码

常用编码：

  • ASCII: 数字、英文字母、英文标点等的编码，每个字符占1字节（8比特）
  • GB2312,GBK：国内爱用的编码，兼容ASCII，汉字用两字节表示

  • UTF-8: 使用较为广泛的编码，兼容ASCII，与GBK等冲突，汉字一般使用三字节表示

个人经验而言，国内Windows电脑使用gbk编码较多，其他设备多是使用utf8编码。utf8能编码的字符比gbk多

2. C语言语法相关

2.1. 操作顺序

  • 对于表达式 (a,b,c) ，a,b,c会顺次执行，最后返回c的值
  • 自增自减的 a++ 与 ++a 两种形式的区别在于将它们传参时是返回操作前或操作后的值

  • 对于表达式 (a && b) ，b会在a非0时执行，最后返回bool值而非a或b的计算值（别python病犯了）

2.2. 复杂类型定义与理解

对于复杂类型（函数指针、指针、数组复合结构），优先找括号，其次向右看，最后向左

  • E.g. char *ptr[10] 是指针数组（先是长度为10的数组，再是内部元素类型为指针） (sizeof(ptr)=80, sizeof(*ptr)=8)
  • E.g. int (*ptr)[10] 是数组指针（先是指针，再是其指向内容为长度10的int型数组） (sizeof(ptr)=8, sizeof(*ptr)=10)
  • E.g. int (*ptr[10])(char*,char*) 是函数指针数组（基本等同于第一个例子只不过内部元素为函数指针）使用类似 ptr[num]("...", "..."); 进行函数调用

观察下来汉语里从左到右顺序对应的类型顺序大概是从里到外（从访问角度来看的里外），优先级从低到高

问ai：带有参数且返回 (函数指针 数组)指针的 函数指针 该怎么定义？

方法一：使用typedef逐步定义类型

// 步骤1：定义基础函数指针类型（指向返回 int，接受两个 int 的函数）
typedef int (*FuncPtr)(int, int);

// 步骤2：定义指向“包含5个 FuncPtr 的数组”的指针类型
typedef FuncPtr (*ArrayPtr)[5];

// 步骤3：定义指向函数的指针，该函数接受 (float, char)，返回 ArrayPtr
typedef ArrayPtr (*MyFuncPtr)(float, char);

// 使用类型别名定义变量
MyFuncPtr fp;

方法二：直接声明（复杂，但展示语法规则）

int (*(*(*fp)(float, char))[5])(int, int); /* 说明：
int (*                        )(int, int); -> 函数指针
                           [5]  -> 数组
      (*                  )     -> 指针(下面的函数指针的返回值)
        (*  )(float, char)      -> 函数指针(用fp(0.1,'6')调用)
          fp
自下而上为从外到内 */

测试样例：

#include <stdio.h>

#define p(x) { \
	printf("%3d: %20s => %-3ld\n", __LINE__, "sizeof("#x")", sizeof(x)); \
	if(0) printf("%u", (x)); \
}

int main(void)
{
	int (*(*(*fp)(float, char))[3])(int, int) = NULL;
	p(fp);
	p(fp(0,0));
	p(*fp(0,0));
	p((*fp(0,0))[0]);
	p((*fp(0,0))[0](0, 0));
}

2.3. 宏定义

特殊宏：

  • __LINE__ 代码当前行数（应用参照上面的一个示例）
  • __FILE__ 编译时的代码文件名
  • __FUNCTION__ 编译时的代码所在的函数名

特殊字符：

  • # 能将宏传入的参数转换成字面量字符串(用引号括起来并对内部的特殊字符转义)

    #define marco(x) printf("%s = %d\n", #x, (x))
void main(){ int yee=2; marco(++yee); }

    打印出来的结果应该是 yee++ = 2

  • ## 衔接符号（两不同符号合并在一起变成第三个符号）

3. ELF相关

3.1. 反编译

对于ELF文件，可以使用objdump工具执行反编译操作（只能反编译为汇编）：

$ objdump -dS path_to_elf

其中-S选项作用为若ELF文件含有源代码信息且原代码存在则插入源代码作注释
